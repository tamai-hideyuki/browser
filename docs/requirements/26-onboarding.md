# 26. オンボーディング 要件

## 概要
初回起動時の体験。Arc は強烈なオンボーディングを差別化要素にしている。本プロダクトでも、ユーザーが「メインブラウザに置き換える」決断を後押しするフローを設計する。

## 設計原則
- スキップ可能（強制しない）
- 1 ステップは 30 秒以内で完了
- 戻る・進むが自由
- 完了状態を永続化、再表示は明示操作のみ

## フロー全体

```
[1] ようこそ
   ↓
[2] 名前・アイコン入力
   ↓
[3] テーマカラー選択
   ↓
[4] インポート元選択
   ↓
[5] インポート実行
   ↓
[6] デフォルトブラウザ設定
   ↓
[7] 機能ツアー（任意）
   ↓
[8] 完了
```

## 各ステップ詳細

### [1] ようこそ
- アプリ名・ロゴ・キャッチコピー
- 「はじめる」ボタン
- 「すでに使っている：ログイン」（M3 以降）

### [2] 名前・アイコン
- 表示名（同期時にデバイス識別に使用、M3）
- アイコン（絵文字 1 つ or アバター）
- スキップ可

M1 簡略化：このステップは省略可能（プロファイル機能が M2 以降のため）。

### [3] テーマカラー
- グラデーションのプリセット 6〜8 種から 1 つ選択
- 選択した色は最初の Space のテーマに反映
- 「あとで変更できます」のヒント

### [4] インポート元選択
- 検出した既存ブラウザを自動表示：
  - Arc / Chrome / Edge / Brave / Firefox / Safari
- それぞれ「ブックマーク」「履歴」「Cookie/ログイン情報」「パスワード」のチェックボックス
- 「インポートしない」も選択可

検出方法：
- 各ブラウザのデフォルトプロファイルパスの存在確認
- macOS / Windows / Linux のパスを定数化

### [5] インポート実行
- プログレスバー
- 完了後にサマリ表示：「ブックマーク 245 件、履歴 12,403 件をインポートしました」
- 失敗時は詳細ログを「詳細を表示」で確認可

詳細仕様は [21-bookmarks-history.md](21-bookmarks-history.md) のインポート節を参照。

### [6] デフォルトブラウザ設定
- 「このブラウザをデフォルトに設定する」推奨ボタン
- 「あとで」スキップ可
- macOS：システムダイアログを起動
- Windows：設定アプリを開いて誘導（OS 仕様により直接設定不可）

詳細は [25-os-integration.md](25-os-integration.md)。

### [7] 機能ツアー（任意）
重要な独自機能を 4 ステップで紹介：
1. **サイドバー**：左に固定。`Cmd+S` で開閉
2. **コマンドバー**：`Cmd+T` でなんでも検索
3. **Space**：用途別に作業空間を分ける
4. **タブの自動アーカイブ**：開きっぱなしを自動整理

各ステップは：
- 短いアニメーション or スクリーンショット
- 1 文の説明
- 「次へ」「スキップ」

### [8] 完了
- 「準備完了」メッセージ
- 「最初の Space を開く」で通常 UI へ遷移
- 初期 Space には「ようこそページ」（about:welcome）を Pinned に追加

## ようこそページ（about:welcome）
通常 UI 起動後に最初の Pinned タブとして開く：
- ショートカットチートシート
- 公式ドキュメント / フィードバックリンク
- 「ツアーをやり直す」ボタン

## 完了状態の永続化
```typescript
// settings table
{
  "onboarding": {
    "completed": true,
    "completedAt": 1730000000000,
    "version": 1
  }
}
```

## 再起動時の挙動
- `completed: true` なら通常起動
- `completed: false` ならオンボーディング再開（途中状態を復元）
- 設定 → 詳細 →「オンボーディングをやり直す」で再起動可

## アップデート時の What's New
新バージョンに「目立つ新機能」がある場合のみ：
- 起動時にモーダル表示
- 「閉じる」「詳しく見る」
- バージョン番号を `seenWhatsNew: "1.2.0"` で永続化、同バージョンは二度と出さない

頻度の上限：メジャー / マイナー更新のみ。パッチ更新では出さない。

## エンプティステート
オンボーディング完了直後の通常 UI が「がらんどう」に見えないように：
- インポートしなかった場合：about:welcome + 推奨サイトの提案（Pinned に 3〜4 個）
- インポートした場合：インポート済み Pinned が並ぶ

## アクセシビリティ
- すべてキーボード操作可
- スクリーンリーダー対応
- 色覚多様性に配慮（色だけで情報を伝えない）

## 国際化
- 日本語 / 英語（M1）
- 文字列は i18n リソースに分離

## デザイン要件
- フルスクリーンモーダル（既存ブラウザシェル UI を覆う）
- アニメーション：CSS transform / opacity
- 各ステップの遷移は 200ms

## 技術実装
- shell renderer 内の専用ルート（Routes として）
- ストアは onboardingStore（セッション内のみ、最終結果は settings に保存）
- インポートはメインプロセスで実行、進捗は IPC で通知

## A/B テスト・最適化
M3 以降：
- 完了率を計測
- 各ステップでの離脱率を可視化
- ステップごとの所要時間

M1 ではテレメトリ ON 時のみ簡易計測。

## 非機能要件
- オンボーディング起動：500ms 以内
- インポート進捗の更新は 200ms 単位で滑らかに
- 全ステップ完走の所要時間目安：60〜120 秒

## エラー処理
- インポート失敗：スキップ可能、後で再試行可能
- ネットワーク不要（インポート以外）
- DB 失敗：エラー画面 + サポート連絡先

## スコープ外
- アカウント作成（M3 以降）
- AI 機能の紹介（M2 以降のオンボーディングで追加）
- チュートリアルゲーミフィケーション（バッジ等）
