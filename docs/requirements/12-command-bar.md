# 12. コマンドバー 要件

## 概要
`Cmd+T` で起動する統合入力UI。URL入力・検索・タブ移動・各種アクションを単一インターフェースに集約する。Arc の Command Bar / Spotlight / Raycast に類似。

## 起動・基本動作
- ショートカット：`Cmd+T`（新規タブ起動兼用）
- ESC で閉じる
- ウィンドウ中央上部にオーバーレイ表示、背景はぼかし
- 入力は常に空で起動。前回の入力は保持しない

## 入力解釈の優先順位
入力文字列に対して以下の順で候補を生成：

1. **オープン中タブの一致**（タイトル / URL 部分一致）
2. **アーカイブ / 履歴の一致**
3. **ブックマークの一致**
4. **アクションコマンド**（`/clear-history` 等、`/` プレフィクス）
5. **URL として有効**（`http://`, `https://`, `localhost`, `IP`, ドメイン形式）
6. **検索クエリへフォールバック**（デフォルト検索エンジンへ）

## 候補表示
- 最大 8 件
- セクション見出しでグルーピング（Open Tabs / History / Bookmarks / Actions / Search）
- 各候補にファビコン・タイトル・URL（または説明）
- 矢印キー / `Ctrl+N` `Ctrl+P` で移動、Enter で実行
- `Tab` キーで候補のURLを入力欄にコピー（さらに編集可能）

## アクション一覧（最低限）
| コマンド | 機能 |
|---|---|
| `/new-space` | 新規 Space 作成 |
| `/switch-space [name]` | Space 切替 |
| `/clear-history` | 履歴クリア |
| `/pin` | アクティブタブをピン留め |
| `/archive` | アクティブタブをアーカイブ |
| `/duplicate` | アクティブタブを複製 |
| `/copy-url` | URL コピー |
| `/find` | ページ内検索（`Cmd+F` の代替） |
| `/screenshot` | スクショ撮影 |
| `/boost` | 現在のサイトに Boost を適用 |
| `/ask` | AI に質問（`70-ai-features.md`） |

アクションは拡張可能なレジストリで管理。プラグイン式で追加できる構造にする。

## URL 判定ロジック
入力が URL かどうかの判定基準：
- スキーム付き（`http://`, `https://`, `file://`, `about:`）→ URL
- スキームなしでドット含む（`example.com`）→ URL（`https://` を補完）
- スキームなしでドットなし `localhost` / IP → URL
- スペース含む → 検索クエリ
- それ以外 → 検索クエリ

## 検索エンジン
- デフォルト：Google
- 設定で {Google, DuckDuckGo, Bing, Brave, Kagi, カスタム} から選択
- カスタム：`https://example.com/search?q=%s` 形式

## キーボードショートカット内の挙動
- `Cmd+T`：新規タブモード（URL入力で新規タブを開く）
- `Cmd+L`：アクティブタブのURLを編集（既存タブで遷移）
- `Cmd+Shift+T`：直近アーカイブ復元（コマンドバーは出さない）

## 非機能要件
- 起動から入力可能まで 50ms 以内
- 候補生成は 100ms 以内（履歴 100 万件規模で）
- 履歴・タブのインデックスは事前に SQLite FTS5 で構築

## アクセシビリティ
- 候補のフォーカスは VoiceOver で読み上げ
- すべての操作はキーボード単独で完結

## スコープ外
- 自然言語コマンド（「明日の予定を開いて」等は AI 機能側で対応）
- マクロ・ワークフロー（フェーズ3 検討）
