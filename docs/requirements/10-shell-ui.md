# 10. シェルUI 要件

## 概要
ブラウザの「枠」となるUI。ウィンドウ、サイドバー、タイトルバー、テーマシステムを担当する。タブ・コマンドバー等の機能別UIは別ドキュメント。

## ウィンドウ
### 機能要件
- フレームレスウィンドウ。macOSのトラフィックライト（赤・黄・緑）はネイティブ位置に表示
- ウィンドウサイズ・位置は終了時に永続化、次回起動時に復元
- マルチウィンドウ対応（`Cmd+N` で新規ウィンドウ）
- ウィンドウごとに独立した Space を選択可能
- フルスクリーンモード対応
- ウィンドウ最小サイズ：横 720px / 縦 480px

### 非機能要件
- ウィンドウ起動から最初のレンダリング完了まで 1.5 秒以内（M2 目標）
- リサイズ時の描画ジャンクなし

## サイドバー
### レイアウト
```
┌─────────────────────────┬───────────────────────┐
│ [Space切替]             │                       │
│ ─────────────────────── │                       │
│ [Pinned Tabs]           │                       │
│   ・ Gmail              │     Web Content       │
│   ・ Calendar           │     (アクティブタブ)  │
│ ─────────────────────── │                       │
│ [Today Tabs]            │                       │
│   ・ ...                │                       │
│ ─────────────────────── │                       │
│ [+ New Tab]             │                       │
└─────────────────────────┴───────────────────────┘
```

### 機能要件
- 幅：240px デフォルト、180〜400px の範囲でリサイズ可能
- ドラッグハンドルで幅調整、ダブルクリックでデフォルトに戻す
- `Cmd+S` でサイドバー表示/非表示トグル
- 非表示時はホバーで一時的にオーバーレイ表示
- 上から順に：Space切替 / Pinned / Today / 新規タブボタン
- セクション区切りは視覚的に明確に

### 非機能要件
- 100 タブ表示時もスクロール 60fps
- 仮想スクロール採用（タブ数が増えても DOM ノード数は一定）

## タイトルバー / トラフィックライト
- macOS：システム標準のトラフィックライトを画面左上に配置
- Windows / Linux：カスタム閉じる・最小化・最大化ボタンを画面右上に配置
- ドラッグエリア：サイドバー上部の Space 名表示部分

## テーマシステム
### 機能要件
- ライト / ダーク / システム連動 の3モード
- Space ごとにテーマカラー（グラデーション）を設定可能
- テーマカラーはサイドバー背景・アクセントに反映
- 透過効果（macOS のヴィブランシー）対応

### カスタマイズ
- 単色 / 2色グラデーション / 3色グラデーション
- プリセット 10 種以上を提供
- カラーピッカーで任意色を指定可能

### 非機能要件
- テーマ切替時のフラッシュなし（CSS 変数で動的切替）
- システムテーマ追従はリアルタイム

## アクセシビリティ
- VoiceOver / NVDA でサイドバー全要素を読み上げ可能
- すべての操作はキーボード単独で完結
- WCAG AA 準拠のコントラスト比

## 国際化
- フェーズ1：日本語 / 英語
- 文字列は `i18n` ファイルに分離。コード内ハードコード禁止
- RTL（右書き）言語：フェーズ2

## スコープ外
- ウィンドウタイリング（OSに委譲）
- 完全なテーマ自作機能（CSSフルカスタマイズはBoosts側）
