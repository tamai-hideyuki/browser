# 13. キーボードショートカット 要件

## 設計方針
- macOS 標準ブラウザショートカットとの互換性を最優先
- Arc 独自のショートカットを参考にしつつ、衝突は Arc 側を尊重
- すべてのショートカットはユーザーがカスタマイズ可能（フェーズ2）

## デフォルトショートカット一覧

### ナビゲーション
| ショートカット | 機能 |
|---|---|
| `Cmd+T` | コマンドバーを開く（新規タブモード） |
| `Cmd+L` | URLを編集（コマンドバー） |
| `Cmd+W` | アクティブタブを閉じる（アーカイブ行き） |
| `Cmd+Shift+W` | ウィンドウを閉じる |
| `Cmd+N` | 新規ウィンドウ |
| `Cmd+Shift+N` | プライベートウィンドウ |
| `Cmd+Shift+T` | 直近アーカイブ復元 |
| `Cmd+R` | リロード |
| `Cmd+Shift+R` | キャッシュ無視リロード |
| `Cmd+[` / `Cmd+]` | 戻る / 進む |
| `Cmd+F` | ページ内検索 |

### タブ移動
| ショートカット | 機能 |
|---|---|
| `Cmd+Option+→` / `←` | 次/前のタブ |
| `Cmd+1`〜`Cmd+9` | Space 切替（1〜9 番目） |
| `Cmd+Option+1`〜`9` | 現在 Space の N 番目のタブ |
| `Ctrl+Tab` / `Ctrl+Shift+Tab` | タブ送り（macOS Safari互換） |

### ウィンドウ・UI
| ショートカット | 機能 |
|---|---|
| `Cmd+S` | サイドバー表示トグル |
| `Cmd+Option+S` | サイドバー幅リセット |
| `Cmd+Shift+C` | コマンドバー（Cmd+T と同じ） |
| `Cmd+Ctrl+F` | フルスクリーン |
| `Cmd+0` | ズームリセット |
| `Cmd++` / `Cmd+-` | ズームイン / アウト |

### Space / タブ管理
| ショートカット | 機能 |
|---|---|
| `Cmd+Option+N` | 新規 Space |
| `Cmd+D` | お気に入り（Pinned）に追加 |
| `Cmd+Shift+D` | フォルダに追加 |
| `Cmd+Shift+A` | アーカイブ検索 |

### 開発者
| ショートカット | 機能 |
|---|---|
| `Cmd+Option+I` | DevTools |
| `Cmd+Option+J` | コンソール |
| `Cmd+U` | ソース表示 |

### AI（`70-ai-features.md`）
| ショートカット | 機能 |
|---|---|
| `Cmd+Shift+H` | ページ要約 |
| `Cmd+/` | Ask on Page |

## カスタマイズ
- 設定 → キーボードショートカット で全項目を編集可能
- リセット機能あり
- 衝突時は警告（同一ショートカットを複数機能に割当不可）
- 設定は JSON で `~/Library/Application Support/<app>/shortcuts.json` に保存

## プラットフォーム差分
- Windows / Linux：`Cmd` → `Ctrl`、`Option` → `Alt` で読み替え
- システム予約ショートカットとの衝突は OS 側を優先（`Cmd+Space` 等）

## 非機能要件
- ショートカット入力からアクション開始まで 16ms 以内
- カスタム設定の読み込みは起動時のみ。実行時は再読込不要

## スコープ外
- マクロ（複数アクションの連結）
- コンテキスト依存ショートカット（フォーカス位置による分岐）はフェーズ2
