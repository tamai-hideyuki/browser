# 60. パフォーマンス 要件

## 目標値（M2 完了時点）
| 指標 | 目標 | 計測方法 |
|---|---|---|
| コールドスタート（ウィンドウ表示まで） | < 1.5s | macOS Activity Monitor 計測 |
| ホットスタート | < 500ms | 再起動間隔 1分 |
| 新規タブ生成 → 表示開始 | < 100ms | 内部計測 |
| タブ切替 | < 100ms | 内部計測 |
| サイドバー描画（200 タブ） | 60fps | DevTools |
| メモリ：起動直後 | < 300MB | Activity Monitor |
| メモリ：タブ 50 アクティブ | < 4GB | 同上 |
| アイドル時 CPU | < 1% | 同上 |

## メモリ最適化
### タブ Discard（スリープ）
- アクティブでない `today` タブを一定時間後に discard
- discard 時にレンダラプロセス終了 → メモリ解放
- メタ情報（URL / タイトル / スクロール位置 / フォーム未入力）はメインプロセスに保持
- 再アクティブ化時に自動復元
- discard 閾値：デフォルト 1 時間（設定で {15分, 1時間, 12時間, 無効}）

### Pinned タブの扱い
- Pinned はデフォルトで discard 対象外（ユーザー期待値優先）
- 設定で「Pinned も discard する」オプションあり

### メモリプレッシャー対応
- OS のメモリ警告を購読
- 警告受信時：非アクティブタブから順に強制 discard
- 続いて、利用頻度の低いキャッシュをパージ

## 起動最適化
- シェル UI は最小限の状態で先行表示
- Space / Pinned の復元はバックグラウンドで段階的に
- WebView の生成は遅延（実際にアクティブ化されたタブのみ）
- 初回起動と通常起動を区別し、通常起動は前回状態を即時復元

## レンダリング最適化（シェル側）
- サイドバーは仮想スクロール
- アニメーションは CSS transform / opacity に限定（layout / paint を発生させない）
- ファビコン読み込みは IntersectionObserver で遅延
- React / Solid どちらでも、再レンダリング範囲を最小化

## ネットワーク
- HTTP/2 / HTTP/3 は Chromium デフォルト
- DNS over HTTPS：`50-security-privacy.md`
- プリフェッチ：DNS のみ。リソースプリフェッチは無効（バンド幅とプライバシー）

## ディスク IO
- SQLite は WAL モード
- 書き込みは debounce（タブ位置変更等は 500ms バッファ）
- 起動時 DB 接続は遅延（最初のクエリ発行まで開かない）

## GPU
- WebGL / WebGPU は Chromium に委譲
- シェル UI は CSS のみで GPU レイヤを利用
- 高負荷ページのフレーム落ちはシェルに伝播させない（プロセス分離）

## 計測・監視
### 内部計測
- 主要操作のレイテンシを記録（タブ切替、起動、検索）
- ローカルに 7 日分の統計を保持
- `about:debug` で表示

### テレメトリ（オプトイン）
- ユーザー同意のもと匿名化して送信
- 送信内容は `90-update-telemetry.md`

## ベンチマーク・回帰検出
- CI で起動時間 / メモリのベンチを毎日実行
- 5% 以上の悪化で警告
- `Speedometer` / `Jetstream` 等の標準ベンチも参考値として計測

## バッテリー / 省電力
- バックグラウンドタブの `requestAnimationFrame` 抑制（Chromium 既定）
- 動画自動再生は Pinned 含めバックグラウンドで停止
- macOS の Low Power Mode 検出時：discard 閾値を短縮

## 非機能要件まとめ
- 100 タブ常時保持の運用に耐える
- 8GB RAM Mac でメインブラウザ運用可能
- ノートPC バッテリー駆動 8 時間ブラウジングで Chrome と同等以上

## スコープ外
- 独自の JIT 最適化（V8 利用）
- カーネルレベルチューニング
