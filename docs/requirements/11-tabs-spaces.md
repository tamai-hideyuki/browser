# 11. タブ / Spaces 要件

## 概念モデル
```
Profile
 └─ Space (×N)
     ├─ Pinned Tabs   : 常駐。閉じてもアーカイブ行き
     ├─ Today Tabs    : 一時タブ。N日後に自動アーカイブ
     └─ Folder        : Pinned/Todayをグループ化
Archive (Profile横断)
```

## タブのライフサイクル
| 状態 | 説明 | 遷移 |
|---|---|---|
| `today` | 通常の開きっぱなしタブ | 新規作成時のデフォルト |
| `pinned` | サイドバー上部に固定 | ユーザーがピン留め |
| `archived` | 閉じた / 自動アーカイブされたタブ | `today` から N 日後 or ユーザー操作 |
| `closed-permanently` | 完全削除 | アーカイブから手動削除 |

## タブ機能要件
### 基本操作
- 新規タブ：`Cmd+T` でコマンドバー起動 → URL入力で生成
- 閉じる：`Cmd+W`
- 復元：`Cmd+Shift+T`（アーカイブから直近を復元）
- 並び替え：ドラッグ&ドロップ
- 複製：右クリック → 複製
- ピン留め / 解除：右クリック or ドラッグでセクション間移動
- ミュート：タブ右側のスピーカーアイコン
- お気に入り（Favorite）：Pinnedの中でも特別なグローバル固定枠（全Spaceで共有）

### タブのプレビュー
- ホバーで小さなプレビュー（ファビコン + タイトル + URL）
- アクティブタブはサイドバーで強調表示

### 自動アーカイブ
- デフォルト：12 時間操作のなかった `today` タブを自動アーカイブ
- 設定で {1h, 12h, 24h, 7d, 30d, 無効} から選択
- Pinned タブは対象外
- アーカイブ時にメモリ解放（タブの`discard`）

### Split View（分割表示）
- 同一Space内のタブを並べて表示（最大 4 タブ）
- ドラッグでウィンドウ内に並べる
- それぞれ独立した WebView インスタンス

## Spaces 機能要件
### Space の構成要素
- 名前
- アイコン（絵文字 1 つ or アイコン）
- テーマカラー（gradient）
- 関連プロファイル（`40-profiles-sync.md`）
- Pinned タブ群
- Today タブ群

### 操作
- 切替：サイドバー上部のドット or `Cmd+1`〜`Cmd+9`
- スワイプ切替：トラックパッド 2 本指横スワイプ（macOS）
- 新規 Space：`Cmd+Option+N`
- Space 並び替え：設定画面でドラッグ
- Space 削除：確認ダイアログ後、内部のタブはアーカイブへ

### 制約
- Space あたりタブ上限なし（パフォーマンス的には 200 を実用上限の目安）
- 1 ユーザーあたり Space 数上限なし（実用上 20 程度）

## Archive 要件
- 全 Space のアーカイブを横断検索
- 検索：タイトル / URL / 訪問日
- 復元：元の Space に戻すか、現在の Space に追加するか選択
- 完全削除：個別 / 期間指定一括
- 容量管理：30 日経過したアーカイブは自動完全削除（設定可）

## Little Arc
- 外部アプリから開かれたリンクを軽量ポップアップで表示
- ポップアップサイズ：800×600（リサイズ可）
- 「タブとして残す」ボタンで通常タブに昇格
- 閉じると履歴のみ残る（タブとしては残らない）
- デフォルトブラウザ設定時の挙動

## 非機能要件
- タブ切替時の表示切替は 100ms 以内
- 100 Pinned + 100 Today = 200 タブ保持時、サイドバーのスクロールは 60fps
- アーカイブされたタブはメモリを保持しない（タイトル・URL・ファビコンのみメタとしてDB保存）

## スコープ外
- タブのクラウド同期（フェーズ3）
- AI 自動グルーピング（`70-ai-features.md` で扱う）
- 他ユーザーとのタブ共有
