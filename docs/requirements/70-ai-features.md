# 70. AI 機能 要件

## 方針
AI は「あれば嬉しい」ではなく差別化要素として組み込む。ただしフェーズ1 は最小限に絞り、フェーズ2 以降で拡張する。

## フェーズ1 機能
### 1. ページ要約（Summarize）
- ショートカット：`Cmd+Shift+H`
- 動作：アクティブタブの本文を抽出 → LLM へ送信 → 要約をサイドパネルに表示
- 抽出は Mozilla Readability ライブラリ等を利用（記事系に強い）
- 要約レベル：{TL;DR / 段落要約 / 章ごと要約} を選択

### 2. Ask on Page
- ショートカット：`Cmd+/`
- 動作：ページ本文をコンテキストとして質問入力 → 回答を表示
- ページ全文がコンテキスト窓に収まらない場合は RAG（ベクター検索）で関連箇所のみ送信
- 引用元はページ内ハイライトでリンク表示

### 3. リンク先プレビュー要約
- リンクホバー時、Option キーで「このページの要約を取得」ボタン表示
- バックグラウンドで取得 + 要約 → ツールチップに表示

## フェーズ2 機能
### 4. タブの自動グルーピング
- AI に Today タブを分析させ、フォルダ分けを提案
- ユーザーが承認したら適用

### 5. AI 検索（コマンドバー統合）
- コマンドバーで質問形式の入力を検出 → AI 回答を直接表示
- 回答内のリンクから関連ページへ遷移

### 6. ライティング支援
- テキストフィールド選択時に「校正」「翻訳」「言い換え」アクション

### 7. ページ翻訳
- 全文翻訳（DeepL / OpenAI 等の API 利用）
- 部分翻訳（選択範囲のみ）

## モデル選択
### バックエンド候補
| 提供元 | モデル | 用途 |
|---|---|---|
| Anthropic | Claude（Sonnet/Haiku） | 要約・Ask on Page・推論 |
| OpenAI | GPT-4 系 / GPT-5 系 | 同上 |
| ローカル | Ollama 等 | プライバシー優先ユーザー向け |

### 設定
- ユーザーが API キーを設定（BYOK 方針）
- 「Browser同梱の無料枠」（フェーズ3 検討、要バックエンド）
- ローカルモデル選択時はオフラインで動作

### デフォルト
- 初回起動時はローカルモデル / 無料枠（フェーズ3）を提案
- API キー入力で外部モデルへ切替可能

## プライバシー
- ページ本文を外部 API に送信する機能はすべて **明示的なオプトイン**
- 「機密タブ」マーク：マークしたタブは AI 機能を無効化
- ローカルモデル選択時は外部送信ゼロ
- 送信前にプロンプトとデータをユーザーに表示するオプション

## UI
### サイドパネル
- AI 応答はメインのアクティブタブの右側にサイドパネル表示
- サイドパネルは閉じる・幅変更可能
- 履歴：AI とのやり取りはタブごとに保持（タブを閉じると消える）

### インライン
- Ask on Page はページ内オーバーレイ
- 引用元はページ内テキストにハイライト

## ストリーミング
- 生成中はトークン単位でストリーム表示
- キャンセル可能

## レート制限・エラー
- API レート制限到達時はわかりやすいエラーメッセージ
- 退避先モデル設定（プライマリ失敗時に セカンダリへ）

## ストレージ
- AI クエリ履歴はオプトインで保存
- 保存時もローカルのみ（同期しない）

## 非機能要件
- ストリーム最初のトークンまで 1 秒以内（API 良好時）
- ローカルモデル使用時はマシン性能に依存（要件としては設定可能性のみ）

## 倫理・透明性
- AI による誤情報の可能性を UI に明示
- 引用元の明示（Ask on Page）
- 「これは AI 生成」のマーク

## 公開課題
- BYOK 必須にするか / 無料枠を提供するか（バックエンドコスト次第）
- ローカルモデル統合手段（Ollama / llama.cpp / MLX）
- 機微情報検出（クレカ番号等を含むページの自動除外）

## スコープ外
- AI による自動タスク実行（エージェント機能）はフェーズ4 以降
- AI 生成画像の埋め込み
