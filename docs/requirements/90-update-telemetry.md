# 90. アップデート / テレメトリ 要件

## 自動更新
### 配布チャネル
- **Stable**：通常ユーザー向け。月次リリース
- **Beta**：先行リリース。週次
- **Dev / Nightly**：日次。実験的機能含む

ユーザーは設定でチャネルを選択可能（デフォルトは Stable）。

### 更新フロー
1. 起動時 + 4 時間ごとに更新サーバへ問い合わせ
2. 利用可能なバージョンがあればバックグラウンドでダウンロード
3. ダウンロード完了後、ユーザーに通知
4. 「再起動して適用」or 「次回起動時に適用」を選択

### サーバ
- macOS：Sparkle 互換 or Tauri / Electron 標準のアップデータ
- Windows：MSI ベース or 同上
- Linux：AppImage の自動更新 or OS パッケージマネージャに委譲

### 署名検証
- 更新パッケージは Ed25519 署名検証必須
- 公開鍵はアプリ同梱
- 失敗時は更新中止 + ログ記録 + ユーザーへ通知

### ロールバック
- 更新後の致命的問題検出時：直前バージョンへ自動ロールバック
- 検出方法：起動後 30 秒以内のクラッシュ × 連続 3 回

### Chromium バージョン
- Electron 採用時：Chromium バージョンは Electron に追従
- 重大な脆弱性発覚時は緊急アップデートを実施
- 最新 Chromium との差分は **2 メジャーバージョン以内** を維持目標

## クラッシュレポート
### 収集対象
- メインプロセスクラッシュ
- レンダラプロセスクラッシュ
- Unhandled exception
- ハング検出

### 内容
- スタックトレース
- アプリバージョン / OS バージョン / アーキ
- クラッシュ時の操作（直近 N イベント）
- ユーザー設定の一部（クラッシュに関連する範囲のみ）

### プライバシー
- **オプトイン必須**（初回起動時に同意取得）
- URL や入力テキストは送信しない
- ユーザーは送信前に内容を確認可能（「詳細を表示」）
- 個人情報フィルタを適用（メールアドレス・トークンらしき文字列の除去）

### 送信先
- 自社収集サーバ（Sentry セルフホスト等）
- 保持期間：90 日

## テレメトリ（利用統計）
### 方針
- **デフォルト OFF**。オプトイン
- 集計用途のみ。個別ユーザーの追跡はしない
- いつでも OFF に切替可能

### 収集対象（ON 時のみ）
| カテゴリ | 内容 |
|---|---|
| 利用 | 起動回数 / セッション長 / DAU |
| パフォーマンス | 起動時間 / メモリ / クラッシュ率 |
| 機能利用 | 各機能の使用回数（コマンドバー / Boosts / AI） |
| エラー | ハンドリングされたエラーの種類 |

### 送信頻度
- 1 日 1 回バッチ送信
- ネット未接続時は次回まで保留

### 匿名化
- インストール ID（ランダム生成）でグルーピング
- IP アドレスは保存しない（geo は国レベルのみ）
- すべての文字列フィールドは事前定義 enum のみ

## ライセンス・コンプライアンス
- OSS 依存ライブラリの一覧を `about:credits` で表示
- ライセンス文書を同梱
- 自動更新のサーバは GDPR / 日本の個人情報保護法に準拠

## 緊急時対応
### 重大脆弱性発覚時
- 全チャネルへ緊急パッチ配信（最短 24 時間以内）
- ユーザーへ通知（アプリ内バナー + メール / 同期アカウント保有時）

### サービス停止
- 更新サーバ停止時もアプリは動作継続
- AI 機能等のオンライン依存機能は graceful degradation

## 透明性
- 設定 → 詳細 → テレメトリ で「直近送信したデータ」を表示
- 公式サイトに「収集データの説明」ページ
- ソースコード公開（フェーズ判断）

## 非機能要件
- 更新確認の通信は < 50KB
- 更新ダウンロードはバックグラウンドのみ。前面 UI に影響しない
- テレメトリ送信失敗はリトライ。失敗続いたらローカル蓄積上限でドロップ

## スコープ外
- ユーザー単位のフィーチャーフラグ配信（フェーズ3 検討）
- A/B テストフレームワーク（同上）
