# 23. メディア再生 要件

## 概要
動画・音声の再生体験。Chromium 標準のメディア機能を最大活用しつつ、ブラウザシェル側で必要な統合（PIP・メディアキー・タブ単位の制御）を提供する。

## サポートする再生形式
- HTML5 `<video>` / `<audio>`
- HLS / DASH（サイト側プレイヤーに委譲）
- WebRTC（音声・ビデオ通話）
- Web Audio API
- DRM（Widevine）：Chromium 標準準拠

コーデックは Chromium に同梱のものに準拠。独自追加はしない。

## タブ単位の音声制御
### インジケータ
- 音声出力中のタブにスピーカーアイコン
- ミュート中：×印付きスピーカー
- サイドバー上のタブ項目に表示

### 操作
- タブ右側のスピーカーアイコンクリックでミュート / 解除
- 右クリックメニュー：「このタブをミュート」
- サイト全体ミュート：右クリック →「このサイトをミュート」（M2 以降）
- すべてのタブをミュート：コマンド `/mute-all`（M2 以降）

### バックグラウンド再生
- バックグラウンドタブも音声再生は継続（Chromium デフォルト）
- 動画の `requestAnimationFrame` は OS により抑制される（バッテリー保護）
- discard 対象判定：音声再生中のタブは discard しない

## Picture-in-Picture（PIP）
### 動作
- 動画要素の右クリックメニューに「ピクチャー・イン・ピクチャー」
- ショートカット未割り当て（Chromium 標準動作）
- ブラウザを切り替えても PIP ウィンドウは残る
- PIP 中にタブを閉じても PIP は再生継続（Chromium 挙動）

### Document Picture-in-Picture API
- 任意 DOM の PIP 化に対応（Chromium 116+）
- M1 ではサイトの実装に委譲

## フルスクリーン動画
- `requestFullscreen` API でフルスクリーン
- ESC で解除
- フルスクリーン中はサイドバーを完全非表示
- Cmd+Ctrl+F とは別動作（後者はウィンドウ全体のフルスクリーン）

## メディアセッション API
サイトが `navigator.mediaSession` で公開するメタデータを利用：
- 曲名・アーティスト・アートワーク
- 再生 / 一時停止 / 次へ / 前へ ハンドラ

利用先：
- macOS の Now Playing（コントロールセンター）
- メディアキー（再生/停止/前後）
- ロック画面（M2 以降検討）

## メディアキー対応
| キー | 動作 |
|---|---|
| Play/Pause | アクティブな MediaSession を持つタブで再生/停止 |
| Next | 同上、次トラック |
| Previous | 同上、前トラック |

衝突時の優先：直近に再生したタブ。

ただしブラウザ側での実装は Chromium に委ねる（独自フックは不要）。OS 統合は [25-os-integration.md](25-os-integration.md) を参照。

## 自動再生ポリシー
Chromium の自動再生ポリシーを踏襲：
- ユーザー操作なしの音声付き自動再生はブロック
- Media Engagement Index で許可サイトは許可
- ユーザーがサイトごとに「自動再生を常に許可」設定可能

## HW アクセラレーション
- デフォルト ON（Chromium 任せ）
- 設定で OFF 可能（トラブル時用）

## 画面共有（getDisplayMedia）
- ピッカー UI を使用（Electron 標準）
- 全画面 / ウィンドウ / タブ から選択
- カスタムピッカーは M2 以降検討

## カメラ / マイク
- 権限要求は [50-security-privacy.md](50-security-privacy.md) のルールに従う
- 利用中はアドレスバー横にインジケータ
- 「最近使ったサイト」を権限管理画面で確認可

## キャプチャ・録画
- 画面録画 API は Chromium 標準
- 独自録画機能は提供しない（M1）

## AirPlay / Chromecast
- M1：未対応
- M2 以降：Chromium のキャストフレームワークに委譲

## サイドバーへの影響
音声再生中のタブが多数ある場合、サイドバー検索フィルタ「音声再生中のみ」を提供（M2）。

## 非機能要件
- 1080p 60fps 動画再生で CPU 占有率 30% 以下（HW 加速時）
- バックグラウンドタブで音楽再生中、フォアグラウンドの体感に影響なし
- PIP の表示開始は 200ms 以内

## 開放的決定事項
- メディアセッションのフォーカス管理ロジック（複数タブ同時再生時の優先順位）
- ロック画面メディアコントロール統合の対応範囲

## スコープ外
- 独自プレイヤーの実装
- DRM の独自実装（Widevine 以外）
- 動画ダウンロード機能
