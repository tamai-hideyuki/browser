# 40. プロファイル / 同期 要件

## プロファイルの概念
プロファイル＝独立した永続化空間。Cookie・履歴・ブックマーク・設定を完全分離する。

```
Profile (Personal)
 ├─ Cookies / Storage
 ├─ History / Bookmarks
 ├─ Settings
 └─ Spaces[]    ← 1プロファイル多Space

Profile (Work)
 ├─ ...
 └─ Spaces[]
```

## フェーズ1：単一プロファイル
- ユーザー1人 = プロファイル1つ
- ローカル保存のみ
- Space は同一プロファイル内で複数作成可

## フェーズ2：複数プロファイル
### 機能要件
- プロファイル作成・削除・名前変更・アイコン設定
- Space ごとにどのプロファイルを使うか選択
- 同一ウィンドウ内で Space 切替時にプロファイルも切り替わる
- プロファイル間はクッキー・ローカルストレージ完全分離

### UI
- 設定 → プロファイル
- 各 Space のプロパティでプロファイル選択
- Space サイドバーにプロファイルアイコン表示

## フェーズ3：クラウド同期
### 同期対象
| 項目 | 同期 | 理由 |
|---|---|---|
| Pinned / Favorite | ✓ | デバイス間で必要 |
| Spaces 構成 | ✓ | |
| 設定 | ✓ | |
| 履歴 | △（オプション）| プライバシー配慮 |
| Boosts | ✓ | |
| Cookie / セッション | ✗ | セキュリティ上同期しない |
| Today タブ | △（オプション）| 「タブ送信」機能で代替 |
| アーカイブ | ✗ | サイズ大、ローカルのみ |

### 同期方式
- **エンドツーエンド暗号化（E2EE）必須**
- 同期キーはユーザーパスフレーズから派生
- サーバはバックエンドストレージ（暗号化済み blob）のみ
- 競合解決：CRDT（Yjs / Automerge 等）または LWW（Last-Write-Wins）

### バックエンド方針候補
1. 自社サーバ（Cloudflare Workers + R2 / D1）
2. ユーザー指定の S3互換ストレージ（BYO）
3. iCloud Drive / Dropbox 経由の同期（OS統合）

第一候補は (1)。ただし運用コストとプライバシー方針で (3) も検討。

## アカウント
- フェーズ3 でメール / パスキーログイン
- アカウントなしでもブラウザ単体で完結（同期はオプション）
- パスキー優先、フォールバックでメール

## デバイス間操作
### Send Tab to Device（フェーズ3）
- アクティブタブを別デバイスにプッシュ
- 受信側に通知

### Continue on Device
- 別デバイスの直近タブを「続きから」で開く

## プライバシー
- アカウント削除時：サーバ上の暗号化 blob を即時削除
- 復号鍵を忘れた場合：リセット可。ただしデータは復号不能
- ログイン履歴・端末リストの可視化

## プロファイル間データ移行
- インポート/エクスポート機能で双方向移行可能
- 競合時はマージ（重複は URL 一致で除去）

## 非機能要件
- プロファイル切替は 500ms 以内
- 同期の差分送信は 5 秒以内（ネット良好時）
- 暗号化処理はバックグラウンドスレッド

## スコープ外
- 組織アカウント / SSO（フェーズ4 検討）
- プロファイル間の Cookie 共有
- サーバ側の生データアクセス機能
