# 22. ダウンロード 要件

## 概要
HTTPレスポンスがダウンロード対象のとき、ファイル保存を管理する。Chromium の DownloadManager をラップしつつ、独自の UI とポリシーを提供する。

## ダウンロード判定
- レスポンスヘッダに `Content-Disposition: attachment` がある
- MIMEタイプが既知バイナリ（`application/octet-stream`, `application/pdf` 設定により等）
- ユーザーが「リンクを保存」を選択
- ユーザーが「画像を保存」を選択

## 保存先
- デフォルト：`~/Downloads`
- 設定で変更可能
- 「毎回確認する」オプションあり
- ドラッグ&ドロップでデスクトップ等へ即時保存可能

## ダウンロードキュー
- 並列上限：6（設定可、1〜10）
- キュー超過分は待機状態
- 各エントリの状態：`queued` / `in-progress` / `paused` / `completed` / `failed` / `cancelled`

## UI
### ダウンロード一覧
- `about:downloads` 画面で全履歴表示
- サイドバー下部にアクティブダウンロード件数のバッジ
- ダウンロード中はサイドバー下部にミニプログレスバー

### 通知
- 完了時：OS 通知 + サイドバーアニメーション
- 失敗時：通知 + エラー詳細リンク

## 操作
| 操作 | 機能 |
|---|---|
| 一時停止 / 再開 | キューエントリの右クリック |
| キャンセル | キューエントリの右クリック |
| Finder で表示 | 完了後のエントリ右クリック |
| 開く | 完了後のエントリダブルクリック |
| 削除（履歴のみ）| エントリ右クリック |
| 削除（ファイルごと）| エントリ右クリック |

## セキュリティ
- 実行ファイル（`.exe`, `.dmg`, `.app`, `.pkg`, `.sh`, `.bat`）は確認ダイアログ
- macOS Gatekeeper の隔離属性は OS の挙動に従う
- スキャン：OS 標準のセキュリティ機構に委譲（独自実装はしない）
- HTTPS 以外からのダウンロードは警告

## 履歴
- ダウンロード履歴は SQLite に保存
- 保存項目：URL / 保存先 / サイズ / MIME / 開始時刻 / 完了時刻 / 状態
- 保持期間：90 日（設定可）
- ファイル本体の存否は表示時に確認（移動・削除されていれば「見つかりません」表示）

## レジューム
- HTTP Range 対応サーバはレジューム可能
- 非対応サーバは再開時に最初から
- ブラウザ再起動後もキューを復元

## プライベートウィンドウ
- ダウンロードは可能
- 履歴には残さない（セッション内のみ表示）

## 非機能要件
- 同時 6 ダウンロード時もシェルUI のレスポンスは維持
- 大容量（10GB+）でもプログレス更新が遅延しない（debounce 200ms）

## スコープ外
- BitTorrent / IPFS 等の独自プロトコル
- ダウンロードマネージャ的な高機能（速度制限、スケジューリング）はフェーズ3
