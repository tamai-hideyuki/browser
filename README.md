# 作成中の気になりポイント
## fetch
HTTPリクエストを送って、レスポンスを受け取る関数。
もともとブラウザのJavaScriptに組み込まれていた機能で、Node.js 18からサーバーサイドでも使えるようになった。
DNS解決、TCP接続、TLSハンドシェイク、リクエスト送信、レスポンス受信をまとめてやってくれる高レベルなAPI。

## process.argv
Node.jsがコマンドライン引数を格納する配列。
`npx tsx src/main.ts https://example.com` と実行した場合:
- `[0]` Node.js本体のパス
- `[1]` 実行したスクリプトのパス
- `[2]` ユーザーが渡した引数（この場合URL）

ほとんどのCLIツールでは `[2]` 以降のユーザー引数だけを扱う。
`[0]` や `[1]` は、エラーメッセージにスクリプト名を含めたい場合などに使うことがある。

## 実行例：
```
npx tsx src/main.ts https://example.com
```
